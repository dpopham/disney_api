Darren's Disney API App - A Work In Progress

Learn more from Darren's Website

There are many great plugins that people have made for accessing the Disney APIs. Some of these include a plugin for node, the themeparks plugin, written by Cubehouse and the python Mousetools module from Scaratozzolo.

However when I wanted to explore the plugins for my own purposes, a limitation I ran into was that I had to keep deep-diving into their internal data structures and processes to figure out the data I needed.

Being the person that I am this did not sit well as I felt there was a simple component missing from these projects that would make them so much nicer to interact with: a restful server implementation.

For this experiment I started with Cubehouse's node implementation and wrapped it into a restful service that I could then reliably interact with. I skipped all the other parks that it supports and focused on only its support for the Disney parks.

There is no right or wrong way as to how to build a RESTful API or the response structures that it returns, there is just opinion. So this experiment is a work in progress subject to change at any time I feel like expressing my opinion of a restful API for such a service. It is kind-of HATEOS in its structure, kind-of unique for some of the API endpoints.

If you like it cool. If not, oh well, go write your own.

If your browser supports parsing of JSON and converting links into clickable objects then interacting with the API directly should work. It is located at Darren's Disney API Test Link: https://disney.darrenpopham.com/api

If your browser doesn't parse JSON into a useable format then I am also working on a Swagger doc located at https://disney.darrenpopham.com/api-docs/
